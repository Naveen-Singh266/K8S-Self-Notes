What ^ Means:
The caret (^) allows compatible minor and patch updates, but not major updates.
means npm can install:
4.18.2   ✅
4.18.3   ✅
4.19.0   ✅
4.20.5   ✅
Node.js uses SemVer (Semantic Versioning):
MAJOR.MINOR.PATCH
Example:
4.18.2
│ │  │
│ │  └── Patch (bug fixes)
│ └──── Minor (new features)
└────── Major (breaking changes)

STEP 4 – Terraform for Kubernetes Deployment
Create folder:
infra/
provider.tf
provider "kubernetes" {
  config_path = "~/.kube/config"
}

main.tf
resource "kubernetes_deployment" "app" {
  metadata {
    name = "devops-app"
    labels = {
      app = "devops"
    }
  }

  spec {
    replicas = 2

    selector {
      match_labels = {
        app = "devops"
      }
    }

    template {
      metadata {
        labels = {
          app = "devops"
        }
      }

      spec {
        container {
          name  = "devops-container"
          image = "localhost:5000/devops-app:1"

          port {
            container_port = 3000
          }
        }
      }
    }
  }
}

resource "kubernetes_service" "app" {
  metadata {
    name = "devops-service"
  }

  spec {
    selector = {
      app = "devops"
    }

    port {
      port        = 80
      target_port = 3000
    }

    type = "NodePort"
  }
}

STEP 6 – Create GitHub Actions Workflow
.github/workflows/pipeline.yml  
In-order to create this.
Go to the repository(devops-ci-cd-project)
Click on Add file > Create new file > paste(.github/workflows/pipeline.yml) in place of the "Name your file".
Update the ci-cd code in pipeline.yml and commit it.


name: Local CI-CD Pipeline

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: self-hosted

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Docker Login
        run: echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin

      - name: Build Docker Image
        run: |
          docker build -t naveen266/devops-app:${{ github.run_number }} .
          docker push naveen266/devops-app:${{ github.run_number }}

      - name: Terraform Init
        working-directory: infra
        run: terraform init

      - name: Terraform Validate
        working-directory: infra
        run: terraform validate

      - name: Terraform Plan
        working-directory: infra
        run: terraform plan -var="image_tag=${{ github.run_number }}"

      - name: Terraform Apply
        working-directory: infra
        run: terraform apply -auto-approve -var="image_tag=${{ github.run_number }}"

      - name: Wait for Deployment
        run: kubectl rollout status deployment/devops-app

      - name: Verify the Pods
        run: kubectl get pods -o wide
		
Go to setting of inside the repository(devops-ci-cd-project) > Secrets and variables > action > New repository secret
Create the DOCKER_USERNAME & DOCKER_PASSWORD save and close it.
